- 系统后端真实数据源已做私有化隔离部署。

## 🎯 项目背景 (Background)
在“一盘货”的多渠道电商业务线中（覆盖天猫、抖音、小红书等），前端销售高度依赖“虚拟套组（组合包）”，导致底层物理大仓库存的核销链路极度复杂。
传统人工盘点模式存在**三大致命痛点**：
1. **核算极耗时**：跨平台连表 VLOOKUP 解析虚拟组合，日均耗时超 3 小时。
2. **错算概率高**：BOM 表级人工乘法换算极易出错，引发超卖危机或工厂冗余生产。
3. **调度严重滞后**：超卖后缺乏“平替”商品的快速寻回机制，且缺乏周转预测，导致潜在 GMV 白白流失或库存积压。

本系统旨在通过**数据清洗管道化**与**监控预警可视化**，将原先 3 小时的人工对账时间压缩至 **3 秒钟**，并赋予系统自动挽回缺货订单的调度能力与精准的补货预测能力。

## 🚀 核心功能 (Core Features)

### 1. ⚙️ BOM表级“虚拟套组”解码引擎
- 自动打通并重组多渠道独立库存报表。
- 通过矩阵级连表运算，将前端复杂的虚拟组合 Code 精准映射并“穿透”至底层物理 Code，实现库存消耗的 100% 准确核算。

### 2. 📈 ADS 动态销量预测与智能备货模型
- **指标全量打通**：无缝打通近期销售流水数据，系统级引入 **ADS (Average Daily Sales, 日均销量)** 与 **DIO (Days Inventory Outstanding, 周转天数)** 核心监控指标。
- **自动化补货中枢**：**彻底抛弃传统的绝对值静态预警**。系统支持动态配置目标备货周期，基于真实日销推演周转健康度，并自动生成各渠道的精准智能补货单，告别拍脑袋备货。

### 3. 🚨 全局 TTL 水位监控与动态预警大屏
- 统筹计算底层可用量、在途量得出全局 TTL（总可用大盘）。
- **动态阈值**：支持全局默认告急线（如 50），并开放交互式数据表支持单品自定义警戒水位。
- **多级透视视图**：采用 Pandas MultiIndex 打造多级数据表头，内置 UI 开关按需折叠/展开“渠道明细指标”，在海量数据展示与视觉整洁度之间达到完美平衡。

### 4. 🔄 智能 NLP “行级联动”平替调度
- **业务痛点击破**：解决缺货后盲目寻找替代品的难题。
- **算法清洗**：内置正则表达式（Regex），自动剔除品牌词及无用业务尾缀（如“样”、“非”、“正装”、“赠品”等）。
- **交互逻辑**：鼠标点击超卖商品行，系统瞬间跨区抓取“同款异码”的闲置物理库存，以抽屉式卡片一键输出替补发货方案。

### 5. 🔍 双向溯源链路与全景检索
- **正向查组合**：输入虚拟 Code，秒速透视其绑定的底层物料“血条”健康度与周转预期。
- **反向查单品**：追踪底层 SKU 在各大渠道的实时消耗分布。
- **模糊查全盘**：支持通过非标准名称（如“睫毛夹”）泛化拉取全库相关物料。
